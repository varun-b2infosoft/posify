Design a cohesive, high-performance POS mobile application UI flow connecting the Dashboard, Sidebar Navigation, and Sell Items (POS) screen.

The experience should feel seamless, fast, and business-focused, allowing users to move from insights → action (selling) in under 2 taps.

🧱 Global App Structure
Use a persistent navigation system
Sidebar (drawer) accessible from all screens
Bottom navigation optional (Dashboard + POS priority)
Maintain consistent design language across all screens
🏠 Screen 1: Dashboard (Entry Point)
Purpose:

Give a quick business overview + fast entry into selling flow

Key UX Addition:
Add a primary CTA button: “Start Selling”
Positioned prominently (top or floating button)
On tap → opens POS screen instantly
Quick Actions Row:
New Sale
Add Product
View Orders

💡 Insight:
Dashboard isn’t just for stats—it should push user into action (POS)

☰ Sidebar Navigation (Global Drawer)
Trigger:
Hamburger icon in header (Dashboard & POS)
Swipe from left
Sidebar Content:
👤 Top Section:
Profile avatar
Shop name
Active shop selector dropdown
📋 Menu Items:
Dashboard
POS (Sell Items) ⭐ (highlighted / most used)
Products
Purchases
Stock Transfer
Invoices
Shops Management
Profile / Settings
💡 Design:
Clean vertical list
Icons + labels
Active screen highlighted with accent color
Slight dark overlay when open
⚡ UX Rules:
Opening sidebar should NOT feel heavy
Tap POS → instant navigation (no delay, no reload feel)
🔄 Transition: Dashboard → POS
Tap:
“Start Selling” OR
POS from sidebar
Animation:
Smooth slide or fade (fast, <200ms)
No dramatic transitions—this is a tool, not a movie

🎯 Core Philosophy:

Speed > Beauty
This is where money happens.

💡 UX:

🔁 Back Navigation (POS → Dashboard)
Back button OR sidebar

🎨 Unified Design System
Colors:

Typography:

⚡ Performance Rules (Non-Negotiable)
Add product: 1 tap
Checkout: ≤ 3 taps
Screen load: instant feel
Animations: <200ms
No blocking loaders
🧠 Smart UX Touches

Highlight “Low Stock” directly in grid

Haptic feedback on add-to-cart
🔥 Core Flow Philosophy

If the user needs to “think” before selling—you’ve already lost.
